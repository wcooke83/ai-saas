Full public site UX copy audit completed March 30, 2026.

**Why:** Comprehensive review of /, /pricing, /chatbot-booking, /sdk, /faq, /help, /wiki, /login, and nav/footer.

**Open issues flagged (not yet fixed):**

1. SDK page: "RAG (Retrieval-Augmented Generation)" appears verbatim in the Custom Chatbots section description — violates terminology convention.
2. FAQ tools section: "How accurate is the AI-generated content?" and "Can I customize the AI output?" are carryover copy from a multi-tool era. VocUI is now chatbot-only; these are off-brand and confusing.
3. FAQ tools section header: Icon is RefreshCw — makes no sense for a chatbot platform. Purely cosmographic but noticeable.
4. Footer "Company" column: Only contains Help and FAQ — duplicates Resources. Missing About, Contact, Careers (linked in header menu but pages don't exist).
5. Pricing page bottom CTA secondary button: "See How It Works" links to /faq — wrong destination. Should link to /#features or /chatbot-booking.
6. Homepage badge: "AI Customer Support, Done Right" — narrow framing; excludes booking and lead capture use cases.
7. Homepage testimonials section heading: "Teams shipping faster with VocUI" — odd; VocUI is a support/booking tool, not a dev productivity tool.
8. Differentiator card: "See exactly where every millisecond goes" — waterfall chart / RAG pipeline telemetry is developer-speak, not appropriate as a top-3 differentiator for a general audience landing page.
9. Chatbot-booking hero CTA button: "Start Free Trial" while the homepage equivalent says "Build Your Chatbot Free" — inconsistent framing across pages.
10. Help page support card for Documentation: description is "Browse our comprehensive guides and tutorials" — overpromises if wiki content is sparse. Button says "View" — too vague.
11. Wiki empty state: "No documentation available yet. Check back soon for guides and tutorials." — this state may appear in production; too apologetic.
12. Login page signup mode description: "Create your first AI chatbot in minutes" — good. Login mode description: "Sign in to access your dashboard" — fine but flat.
13. MFA verify button: "Verify" — acceptable but could be "Confirm Code" for clarity.
14. Help form success state: "We'll get back to you within 24 hours." — good, matches the SLA badge on the email support card.
15. /about, /contact, /careers, /cookies — linked in header mobile menu but pages do not exist (404s).

**How to apply:** Reference this list when prioritizing copy fixes. Items 1, 2, 4, 5 are highest urgency.
