**H1 direction (approved):** Outcome-led, references pain ("stop answering the same questions twice") before the solution ("your knowledge base becomes a chatbot"). NOT product-feature-led.

**Tagline frame:** Deflect support tickets + capture leads + book appointments. These are the three business outcomes the buyer actually cares about.

**ICP:** B2B operations/marketing decision-makers wanting to deflect tickets and automate customer interactions. Secondary: solo founders and agencies (AppSumo distribution).

**Top 3 differentiators (lead with these — competitors don't have all three):**
1. In-chat calendar booking (Easy!Appointments integration — chatbot checks availability and books mid-conversation)
2. Embeddable agent console — live agent handoff that works inside any internal tool via SDK; agents never need to log into VocUI
3. Performance telemetry — Firefox-style waterfall chart with 10+ labeled RAG pipeline stages

**Messaging pillar: "Full loop, not just Q&A"**
Competitors (Chatbase, Tidio, CustomGPT, Crisp) stop at knowledge-base Q&A or basic live chat. VocUI's story is the full customer interaction loop: deflect → capture → escalate → book → analyze.

**What to avoid:**
- Do not lead with "AI-powered" or "custom chatbot" — that's table stakes language in 2024/2025
- Do not use "VocUI" as the object of "trained on" — this was a homepage typo that was corrected
- Never use "best-in-class", "seamless", "cutting-edge"

**Why:** These decisions were made during initial homepage audit/rewrite (March 2026). The hero headline "Build Custom AI Chatbots Trained on VocUI" was identified as a critical bug — VocUI appeared to be the knowledge base rather than the platform.
