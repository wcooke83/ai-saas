Full security audit completed 2026-03-24. 19 issues found (2 critical, 5 high, 5 medium, 5 low).

**Why:** Comprehensive code-level review of all 11 audit domains.

**Critical findings:**
- api-keys.ts debug logging dumps all key hashes on failed lookup (lines 198-209)
- E2E test endpoints (/api/e2e/, /api/auth/e2e-login) not gated on NODE_ENV, only env-var secret

**Key architecture observations:**
- Admin client (service role) is never imported in components -- properly server-only
- getUser() used instead of getSession() for auth -- correct pattern
- API keys are SHA-256 hashed before storage
- Stripe webhook uses constructEvent() correctly with raw body
- Widget markdown renderers escape HTML before transformations (XSS safe)
- CORS is wildcard (*) on ALL API routes including authenticated ones
- AI_MOCK_MODE activates silently when both API keys are missing, even in production
- Monthly message limit check in chat route is TOCTOU (read-then-write without atomicity)
- Several chatbot sub-resource routes use admin client after ownership check via getChatbot()
- user_data and preChatInfo injected into system prompt without sanitization (prompt injection vector)

**How to apply:** Reference these findings when reviewing PRs that touch auth, CORS, RAG prompt building, or admin analytics routes.
